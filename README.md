# 덕질메이트

## 1. 프로젝트 소개

### 프로젝트명

덕질메이트

### 프로젝트 한 줄 소개

캐릭터 굿즈를 쉽고 즐겁게 둘러보고 구매 흐름까지 경험할 수 있는 모바일 쇼핑몰 웹사이트입니다.

### 제작 목적

덕질메이트는 리락쿠마, 짱구, 담곰이, 먼작귀 등 다양한 캐릭터 굿즈를 한곳에서 탐색할 수 있는 쇼핑몰을 목표로 제작했습니다. 사용자가 모바일 환경에서 상품을 빠르게 찾고, 카테고리별 상품 확인, 위시리스트, 장바구니, 상품 상세 페이지, 로그인, 주문 흐름까지 자연스럽게 경험할 수 있도록 구성했습니다.

특히 단순히 상품 이미지를 나열하는 페이지가 아니라 실제 쇼핑몰처럼 보이도록 메인 배너, 신상품, 베스트 상품, 캐릭터별 상품, 랭킹 영역 등을 나누어 사용자의 쇼핑 동선을 설계했습니다.

## 3. 사용 기술

- React
- Vite
- JavaScript (JSX)
- Firebase (Authentication, Firestore)
- Polar SDK (결제/체크아웃)
- Vercel (배포, 서버리스 API Functions)

## 4. 주요 기능

- 메인 배너 슬라이드  
  여러 개의 배너 이미지가 자동으로 전환되며, 사용자가 메인 이벤트 상품을 자연스럽게 확인할 수 있도록 구성했습니다.

- 카테고리별 상품 탐색  
  여름필수템, 전체상품, 신상템, 베스트, 캐릭터, 키링, 생활용품, 차량용품, 폰케이스 등 다양한 카테고리로 상품을 분류했습니다.

- 캐릭터별 상품 보기  
  리락쿠마, 짱구, 담곰이, 먼작귀 등 캐릭터별로 상품을 모아 볼 수 있도록 제작했습니다.

- 상품 상세 페이지  
  상품 이미지, 브랜드명, 상품명, 할인율, 가격, 배송/혜택 정보 등을 확인할 수 있는 상세 화면을 구현했습니다.

- 위시리스트 기능  
  관심 있는 상품을 위시리스트에 담거나 삭제할 수 있도록 구현했습니다. Firebase Authentication으로 로그인한 사용자의 위시리스트/장바구니 데이터는 Firestore의 `savedItems` 컬렉션에 저장되어, 다시 로그인해도 담아둔 상품이 유지되도록 했습니다.

- 장바구니 기능  
  상품을 장바구니에 담고, 수량 변경, 선택 삭제, 위시리스트로 이동, 주문 페이지 이동이 가능하도록 구성했습니다.

- 바로구매 흐름  
  상품 상세 페이지 하단에 관심상품(하트) · 장바구니 · 바로구매 버튼을 고정 배치하고, 바로구매를 누르면 장바구니를 거치지 않고 해당 상품으로 바로 주문/결제 페이지로 이동하도록 구현했습니다.

- 결제 및 결제 결과 화면  
  Polar 체크아웃으로 결제를 진행한 뒤, 결제 결과가 별도의 외부 페이지가 아니라 앱 내부 화면으로 표시되어 뒤로가기나 홈 이동이 자연스럽게 이어지도록 구성했습니다.

- 주문조회 기능  
  결제가 완료되면 상품명, 결제 금액, 결제 상태, 주문일이 주문 내역으로 자동 저장됩니다. 마이페이지의 주문조회 메뉴에서 지금까지의 주문 목록을 확인할 수 있으며, 주문 데이터는 위시리스트와 마찬가지로 Firestore에 저장되어 다시 로그인해도 유지됩니다.

- 검색 기능  
  상품명과 브랜드명을 기준으로 사용자가 원하는 상품을 검색할 수 있도록 구현했습니다.

- 로그인 및 회원가입 화면  
  이메일 로그인, 회원가입, 소셜 로그인 버튼을 포함한 마이페이지 흐름을 구성했습니다.

- 최근 본 상품 기능  
  사용자가 확인한 상품을 최근 본 상품 목록으로 다시 볼 수 있도록 구현했습니다.

- 하단 네비게이션  
  카테고리 · 브랜드 · 홈 · 위시 · 마이 5개 탭으로 구성된 하단 독(Dock) 네비게이션을 만들어, 어떤 화면에서든 주요 페이지로 바로 이동할 수 있도록 했습니다. 현재 위치한 탭은 하이라이트로 표시됩니다.

- 반응형 모바일 화면  
  모바일 쇼핑몰 화면을 중심으로 제작하고, 데스크톱에서는 모바일 기기 프레임처럼 보이도록 레이아웃을 조정했습니다. 모바일에서는 화면 폭을 실시간으로 감지해 어떤 기기에서도 좌우 여백 없이 화면에 꽉 차게 표시됩니다.

## 6. 실행 방법

```bash
npm install
npm run dev
```

- 개발 서버는 Vite 기본 포트(5173)에서 실행됩니다.
- 결제(체크아웃) 기능을 사용하려면 프로젝트 루트에 `.env` 파일을 만들고 아래 환경 변수를 설정해야 합니다.

```text
POLAR_ACCESS_TOKEN=<Polar 샌드박스 액세스 토큰>
```

- `vite.config.js`에 개발용 미들웨어를 구성해 두어, 배포 환경(Vercel)과 동일하게 로컬에서도 `api/` 폴더의 서버리스 함수를 호출할 수 있습니다.

## 7. 문제 해결 과정

### 모바일 화면 비율 문제

처음에는 데스크톱 화면에서 모바일 쇼핑몰 레이아웃이 너무 작거나 크게 보이는 문제가 있었습니다. 이를 해결하기 위해 CSS 변수로 모바일 기준 너비와 확대 비율을 관리하도록 만들었습니다.

다만 처음에는 미디어쿼리로 390px, 410px 같은 고정 폭 구간을 나눠 대응했는데, 구간 사이에 있는 기기에서는 여전히 좌우 여백이 남는 한계가 있었습니다. 그래서 최종적으로는 JavaScript로 화면 폭을 실시간 감지해 뷰 너비와 확대 비율을 동적으로 계산하는 방식으로 전환했고, 어떤 화면 폭에서도 여백 없이 꽉 차게 표시되도록 해결했습니다.

### 상품 데이터가 많아지면서 생긴 관리 문제

카테고리, 캐릭터, 신상품, 베스트 상품 등 여러 영역에서 같은 상품 데이터가 반복적으로 사용되었습니다. 중복 데이터를 줄이기 위해 상품 정보를 객체 형태로 구성하고, 필요한 영역에서 배열로 분류하여 재사용할 수 있도록 정리했습니다.

### 장바구니와 위시리스트 상태 관리

사용자가 상품을 담거나 삭제할 때 화면에 바로 반영되어야 했습니다. 이를 해결하기 위해 상품의 이미지 값을 기준으로 중복 여부를 확인하고, 이미 담긴 상품은 수량을 증가시키거나 위시리스트에서 제거되도록 처리했습니다.

### 모바일 쇼핑몰다운 사용성 구현

단순한 페이지 이동만으로는 실제 쇼핑몰 느낌이 부족했습니다. 그래서 하단 네비게이션, 카테고리 드로어, 최근 본 상품, 상단/하단 이동 버튼, 상품 카드 호버 효과 등을 추가하여 사용자가 더 편하게 둘러볼 수 있도록 개선했습니다.

### 모달이 겹쳐 보이는 문제

상품 빠른보기(퀵뷰) 모달에서 장바구니 버튼을 누르면, "장바구니에 담겼습니다" 확인 모달이 퀵뷰 모달 뒤에 가려져 보이는 문제가 있었습니다. z-index 수치만 조정하는 대신, 장바구니 버튼을 누르는 순간 퀵뷰 모달을 먼저 닫고 확인 모달로 화면이 전환되도록 상태 흐름을 정리해 두 모달이 겹치지 않도록 해결했습니다.

### 결제 흐름 연결

구매하기 버튼 이후의 흐름을 실제 쇼핑몰처럼 만들기 위해 주문 화면과 결제 성공 화면을 구성했습니다. Polar SDK를 이용해 Vercel 서버리스 함수(`api/products.js`, `api/checkout.js`, `api/checkout-session.js`)로 상품 조회, 체크아웃 세션 생성, 결제 결과 확인 흐름을 나누어 관리했습니다. 특히 Polar의 상품 검색 필터가 한글 같은 비영문 텍스트를 안정적으로 매칭하지 못하는 문제가 있어, 상품명으로 체크아웃을 생성할 때는 전체 상품 목록을 가져와 이름을 직접 비교해 상품 ID를 찾도록 처리했습니다.

또한 처음에는 결제 완료 페이지가 쇼핑몰과 분리된 별도 화면으로 떠서 뒤로가기가 되지 않는 문제가 있었습니다. 이를 결제 결과가 앱 내부 페이지로 표시되도록 구조를 바꾸어, 결제 후에도 쇼핑몰 화면 흐름이 끊기지 않고 홈으로 자연스럽게 돌아올 수 있게 개선했습니다. 결제가 확인되면 주문 정보를 Firestore에 저장해 마이페이지의 주문조회에서 다시 볼 수 있도록 연결했습니다.

### Firebase 연동을 통한 로그인/데이터 저장

이메일 로그인과 회원가입 화면만 있고 실제 인증 로직이 없다면 로그인 흐름이 형식적으로만 남게 됩니다. 이를 방지하기 위해 Firebase Authentication으로 실제 이메일/비밀번호 로그인·회원가입을 구현하고, 로그인 상태 변화를 감지해 사용자별 위시리스트 데이터를 Firestore에서 불러오고 저장하도록 연결했습니다.

## 8. 회고

이번 프로젝트를 통해 쇼핑몰 웹사이트는 단순히 예쁜 화면을 만드는 것보다 사용자의 이동 흐름을 설계하는 것이 중요하다는 것을 느꼈습니다. 메인 페이지에서 상품을 발견하고, 상세 페이지에서 정보를 확인한 뒤, 위시리스트나 장바구니로 이어지는 과정을 직접 구현하면서 실제 서비스의 구조를 더 잘 이해할 수 있었습니다.

또한 상품 수가 많아질수록 데이터를 어떻게 정리하고 재사용할지 고민하게 되었고, 반복되는 UI를 컴포넌트처럼 나누어 관리하는 방식의 필요성을 배웠습니다. 모바일 화면에서는 작은 공간 안에 많은 정보를 담아야 하기 때문에 여백, 버튼 크기, 고정 네비게이션, 스크롤 동선까지 세심하게 고려해야 한다는 점도 알게 되었습니다.

Firebase Authentication과 Firestore, Polar 결제 SDK를 연동하면서 프론트엔드만으로는 완성할 수 없는 로그인 상태 관리, 데이터 영속성, 실제 결제 흐름까지 다뤄볼 수 있었던 점도 이번 프로젝트의 큰 수확이었습니다.

결제 완료 후 주문 내역을 Firestore에 저장하고 마이페이지 주문조회에서 다시 확인할 수 있도록 연결하면서, 결제 → 저장 → 조회로 이어지는 데이터 흐름을 처음부터 끝까지 직접 만들어 본 것도 좋은 경험이었습니다.

아쉬운 점은 결제가 아직 Polar 샌드박스 환경에 머물러 있다는 점입니다. 이후에는 실제 결제 연동, 상품 등록/수정 기능, 관리자 페이지, 리뷰 작성 기능 등을 추가하여 더 완성도 높은 쇼핑몰로 발전시키고 싶습니다.
